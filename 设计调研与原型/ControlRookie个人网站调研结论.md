# ControlRookie 个人网站调研结论

## 证据边界

- 已纳入：Codex 全局规则、当前工作区本地产品/技能资产、CSDN 公开页面、`forai.design`、`huasheng.ai`。
- 受限：微信公众号后台链接需要登录态，本轮只能作为“你计划运营公众号”的入口事实，不能直接读取后台内容。
- 工具状态：FrontendDesign 容器健康；Open Design daemon `127.0.0.1:7456` 当前不可达；本机 Chromium 可启动但 CLI 截图落盘异常，因此本轮以静态高保真 HTML 交付视觉方案。

## 个人画像

ControlRookie 的核心画像不是泛技术博主，而是“跟得上时代的 AI 自动化架构师”：

- 工业自动化现场理解深：内容集中在 CodeSys、PLC、MQTT、状态机、Topic Filter、QoS、工程调试与真实交付。
- AI 工具链建设能力强：本地已有 SmartControl CLI、CodeSys LLM Synchronizer、LicOS 技能体系、内容发布工具和 Agent/Skill 工程资产。
- 强验证工程风格明显：全局规则强调真实业务链路、自动化验证、质量门禁、错误自愈、中文交付和路径规范。
- 产品化意识已经形成：工具、技能、程序不只是内部脚本，而是有开源、销售、交付、文档、验收和支持边界的产品雏形。
- 内容增长路径清晰：CSDN 和微信公众号适合作为前期博客入口，后期再迁移到站内文章系统。

一句话定位建议：

> ControlRookie 是面向工业自动化工程师的 AI 自动化架构师，专注把控制、算法、协议、工具链、AI Agent 和工程思想沉淀为可复用的开源内容与产品。

## 参考站拆解

### ForAI

适合借鉴：

- 黑白极简、高级留白、强网格、大字号标题。
- 工作室式作品墙，适合改造成你的“工具 / 技能 / 文章 / 案例”展示墙。
- 横向滚动、悬停反馈、卡片序号、Archive 式索引，适合表达工程资产。
- AI 原生但不浮夸，能避开普通个人博客的廉价感。

不建议照搬：

- ForAI 是设计工作室，不是工业技术个人品牌；文案和案例结构必须替换为工程证据。
- 过度抽象的 AI 视觉会削弱你的“现场可信度”，需要加入 PLC、状态机、协议链、验证门禁等工业语义。

### 花生 / huasheng.ai

适合借鉴：

- 首页先建立人物身份与愿景，再展示内容、产品和项目。
- 个人 IP 叙事效率高，适合你的首屏定位。
- 可用“年份、作品、文章、工具、服务”形成可信指标。

不建议照搬：

- 花生偏内容创业和 AI 产品集合，你的网站需要更硬核、更工程化。
- 视觉上不能过于生活方式化，否则会冲淡工业自动化专业感。

## 推荐方案

推荐采用“个人品牌 + 产品工作室”结构：

1. 首页：一句定位、愿景、代表工程视觉、精选工具/文章入口。
2. 博客：前期跳转 CSDN 和微信公众号；站内只放精选卡片和专题页。
3. 产品：展示 CodeSys LLM Synchronizer、SmartControl CLI、LicOS 技能包、内容/文档/发布工具等。
4. 联系：微信、公众号、CSDN、GitHub、工具购买/合作咨询入口。

视觉方向：

- 基底：ForAI 的界面、风格、配色、字体、导航方式和作品墙组织方式是设计基础，正式稿必须先做高保真还原。
- 个性：工业控制图、梯形图/状态机、扫描线、运行态绿色、警示红。
- 首屏：花生式人物定位效率，但用工程语言表达。
- 动效：不追求复杂滚动和强动画；优先静态页面加简单动画，保证后续维护成本可控。

## 设计验收门禁

当前 `controlrookie-website-design-preview.html` 只能作为风格方向板和信息架构草案，不能作为 ForAI 高保真还原稿。

后续正式设计必须按以下顺序推进：

1. 先只设计一页 ForAI 高保真还原验收稿。
2. 还原重点包括界面结构、风格、配色、字体、导航方式、卡片比例、留白、边框、项目墙组织方式和整体高级感。
3. 动画和滚动效果不是验收重点，可先用静态效果和少量简单 hover / fade / marquee 动画代替。
4. 用户确认这一页的还原逼真度没有问题后，才能继续扩展首页完整内容、博客页、产品页和联系页。
5. 未通过单页验收前，不进入多页面实现和正式前端工程搭建。

## 可行性评估

可行性高。当前项目目录为空，适合从干净的前端项目开始；本地已有 FrontendDesign 环境，可用于后续高保真生成与迭代。

主要难点不在页面实现，而在内容资产治理：

- 产品边界需要明确：免费开源、试用、付费、定制服务要分层。
- 博客入口需要清楚：CSDN、公众号、站内精选之间的关系要统一。
- 微信公众号资料无法公开读取，需要你后续提供可公开的文章列表、二维码或介绍文案。
- 若要实现 ForAI 级别动效，需要正式前端工程，而不是单一静态页。
