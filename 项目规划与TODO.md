# ControlRookie 个人网站项目规划与 TODO

> 单一规划源：本文档是当前项目唯一正式规划、路线图、TODO 和执行记录入口。后续不要再新建同类规划文档，直接更新本文档。

## 项目目标

建设 ControlRookie 个人网站，用 ForAI 的界面、风格、配色、字体、导航方式和作品墙组织方式作为设计基础，表达“跟得上时代的 AI 自动化架构师”的个人定位，并承接文章、产品、工具销售和联系方式。

## 当前技术路线

- 主路线：Open Design 高保真重构。
- 视觉标尺：ForAI 原站截图和必要参数，不直接基于 Framer 编译 HTML 开发。
- 辅助工具：FrontendDesign 仅用于内容密集区的备选方案，不作为首页主原型。
- 验收工具：桌面浏览器实际打开页面，优先 Edge；Playwright/Chromium 仅作桌面辅助截图和自动化检查。
- 正式前端：采用零依赖静态工程承接当前高保真首页，不安装新框架；后续如需 Astro、Next.js 或其它依赖，必须先列出安装路径、用途和预计体积，经用户同意后再执行。

## 核心定位

- 个人品牌：ControlRookie
- 定位：跟得上时代的 AI 自动化架构师
- 目标读者：工业自动化工程师、PLC/CODESYS/SmartControl 工程师、希望用 AI 提升工程效率的技术人
- 核心价值：控制、算法、架构、思想、前沿技术、AI、源码开源分享和工程工具产品化

## 设计总原则

1. ForAI 是设计基础，必须高保真还原其界面气质、配色、字体、导航、网格、留白、卡片和作品墙组织方式。
2. Open Design 版本是当前唯一正式首页高保真原型源。
3. 原站 HTML 只用于调研和参数提取，不进入正式项目产物。
4. 动画和复杂滚动不是必须项，优先静态页面加简单 hover、fade、轻量 marquee 等低维护动效。
5. 工业自动化语义只用于内容和局部视觉，不得破坏 ForAI 的基础审美。
6. 当前 `prototypes/homepage-open-design/index.html` 已被确认为满意的视觉底座基线；后续页面和正式前端工程必须沿用该基线。
7. 网站必须支持中文/英文双语切换，中文界面导航使用中文。
8. 调整正式前端工程时，视觉效果不得漂移；每次结构性修改前必须保存桌面参考图，修改后用同一桌面视口复验。
9. 不采用紫禁城配色点缀，默认延用当前 ForAI 黑白灰和 `#f1f1f1` 浅灰卡片方案。
10. 前端工程代码必须具备工程师级注释，关键控件需标明对应页面、控件用途和后续手动调整入口。
11. 首页、资源、文章、产品、联系等页面必须共享导航栏以下的稳定正文边界；边界是设计和排版约束，不是用户可见装饰，禁止画出可见框线。
12. 左边界必须与左上角 `CR` logo 的左边缘左对齐，右边界必须与右上角 `EN` 语言按钮的右边缘右对齐；上边界参考首页照片上边缘，上下留白保持一致。
13. 正文边界只是最大安全范围，不是贴边排版目标；正文内容不能超出边界，但必须在边界内部保留舒适内边距、呼吸感和视觉秩序，不要求每个页面占满整个边界区域。
14. 灰色卡片只有在可以点击进入文章、产品、资源详情、外部链接或联系方式时才允许 hover 或动画效果；不可点击卡片必须保持静态。
15. 信息类灰色卡片必须保持三段式层级：序号标题、卡片名称、概述；同一行卡片的三段内容起点和垂直间距必须严格对齐，卡片高度由该行内容最多的卡片决定。
16. 全站布局必须以页面垂直中心轴作为核心参照；导航栏居中分布是基准，每个页面主体内容的视觉重心不得明显向左或向右偏移。
17. 首页首屏删除 `AI 自动化架构师 / 2026` kicker，`ControlRookie` 顶部与个人照片顶端稳定对齐，三段主文案之间保持更舒展的垂直间距。
18. 当前首页主体已经由用户确认满意并固定；除非用户明确要求，不得改动首页首屏、照片、主文案、四张领域卡片和对应布局。
19. 首页后的个人经历页在导航中必须命名为 `关于我` / `About Me`，并作为第一个导航按钮；页面必须按 `达姆施达特工业大学 -> Bosch -> Fraunhofer IWES -> 新能源 HIL -> 烟草 -> 冶金传动 -> 控制器厂家` 做清晰可见的时间线表达，不得只做普通卡片堆叠，并预留后续能力描述区域。
20. 文章页和产品页顶部的 `文章`、`产品` 标题必须与主体卡片形成同一内容组，视觉上贴近并从属于卡片区域，禁止标题悬浮在远离卡片的位置。
21. 联系页主文案使用 `工控人太苦了，让我们一起告别水深火热`，删除原说明文案和 `ControlRookie / AI Automation Architect`；主文案不得使用灰色卡片背景，整体布局必须避免上半屏拥挤、下半屏空白。
22. 联系页 Logo 辅助视觉不得使用四角露白或存在白角伪影的位图；优先使用无白角伪影的 SVG 或重新处理后的透明/实底素材，并保持黑白灰底座和二维码可读性。
23. 文章系统必须面向长期大量文章增长，禁止在单一文章入口页无限罗列文章卡片；必须建立分类、系列、标签、专题和子导航。
24. 单篇文章页必须采用左侧固定目录、右侧标题正文、一键到顶/到底、代码块复制按钮、站内互链和评论回复预留区的结构。
25. 文章、资源、产品、源码、标准、模板和工具之间存在参考或引用关系时，必须用超链接快速互跳并保留返回路径。
26. 单篇文章页目录锚点必须唯一、稳定且逐项校验，每个目录链接都必须跳转到对应标题；目录链接不使用正文强调色，正文超链接、面包屑和上一篇/下一篇必须明显可识别。
27. 低饱和琉璃绿可作为链接和品牌点缀色，但不得恢复成大面积绿色装饰，整体仍以 ForAI 黑白灰底座为主。
28. 网站 Logo 必须使用 `E:\07 自媒体\02 Logo\Control Rookie Logo.svg` 最新母版及其最新派生图；禁止使用 `当前在用老Logo` 目录中的旧 Logo，禁止 AI 自行重绘 Logo 图案，网页适配只允许调整背景、尺寸、裁切和摆放方式。
29. 文章目录采用固定专家阅读栏策略：桌面端优先向左侧留白扩展，长章节标题在目录内部自然换行，不为单篇文章临时手工调整宽度；文章以专家身份发布，禁止出现表情、emoji 或娱乐化符号。
30. 文章阅读页目录最终几何规则：不得改变文章主体宽度和位置，不得改变目录右边缘与正文之间的距离；目录只允许左侧向左扩展，宽屏目标到顶部 `ControlRookie` 品牌文字右边缘附近，窄桌面不得越过 Logo 左边缘。
31. 文章页右侧一键到顶/到底按钮必须收在右侧正文安全边界内；目录、正文、滚动按钮都必须用 Playwright 坐标实测校验，不能只凭截图主观判断。
32. 当前文章目录项间距基准为 `.article-toc ol { gap: 9px; }`，目录项前使用小圆点，当前章节使用琉璃绿加粗标识；后续只允许小幅微调。

### 文章系统第一版方案

- 文章入口页从“博客卡片列表”升级为“知识库式文章中心”：顶部提供分类、系列、标签和搜索入口，主体按专题系列、最新文章、分类入口组织，禁止无限纵向罗列。
- 首批以 `CODESYS MQTT 系列` 为样例专题，建立系列进度、文章目录、相关文章和相关资源入口，后续可平滑扩展到 AI 自动化工具、控制算法、工程模板等主题。
- 单篇文章页采用桌面阅读结构：左侧固定目录，右侧为文章标题、元信息、摘要和正文；右侧固定一键到顶、一键到底按钮，整体仍服从当前不可见正文边界和中心轴布局。
- 文章正文排版整体收紧：正文字号比首页小，行高保持舒适，H1/H2/H3、正文、引用、列表和代码块形成清晰层级。
- 代码块统一使用适合代码阅读的等宽字体，顶部显示语言标签和复制按钮，复制后给出明确状态反馈。
- 站内互链必须成为文章系统基础能力：文章、资源、产品、源码、标准、模板和工具之间可直接跳转，并提供返回路径、上一篇/下一篇、同系列文章和相关资源。
