FB_MqttClient 主功能块和状态机怎么跑

源码范围说明

本篇属于 MQTT 源码加更，展示的是 V1.0 运行链路中的核心源码片段、对象职责和阅读顺序；它不是完整工程源码包，也不等同于 1.0 全量源码清单。完整源码资源后续应在资源/源码页统一维护。

MQTT Client 的核心不是 CONNECT 报文，而是状态机。报文只是状态机在某个状态下要做的动作。

状态机主线

Idle -> TcpConnect -> MqttConnect -> Connected 是主路径。Connected 之后才允许发布、订阅、取消订阅和心跳。任何错误都会把状态机推向断开和重连路径。

引脚分区

FB_MqttClient 的输入引脚可以分成五组：连接配置、协议版本、遗嘱、发布、订阅。输出引脚则分成状态、诊断、订阅列表和接收消息。

iecst
FUNCTION_BLOCK FB_MqttClient
VAR_INPUT
    // 连接配置
    bEnable                : BOOL := TRUE;                               // 使能客户端
    bConnect               : BOOL;                                       // 连接命令
    sBrokerIP              : STRING(128) := '192.168.20.222';           // Broker IP 地址
    uiPort                 : UINT := 1883;                               // Broker 端口号
    sClientID              : STRING(128);                                // 客户端标识符
    sUsername              : STRING(128);                                // 用户名
    sPassword              : STRING(128);                                // 密码

    // 协议版本
    eVersion               : E_MqttVersion := E_MqttVersion.byMqttVersion311; // MQTT 协议版本

    // 连接参数
    bCleanSession          : BOOL := TRUE;                               // 是否清理会话
    uiKeepAlive            : UINT := 60;                                 // 心跳周期（秒）
    bUseSSL                : BOOL := FALSE;                              // 是否启用 SSL
    bAutoReconnect         : BOOL := TRUE;                               // 是否自动重连
    uiReconnectDelay       : UINT := 5000;                               // 重连延时（毫秒）

    // MQTT 5.0 连接属性
    udiSessionExpiry       : UDINT := 0;                                 // 会话过期间隔
    uiReceiveMax           : UINT := 65535;                              // 最大接收数量
    udMaxPacketSize        : UDINT := 4096;                              // 最大报文长度
    bRequestResponseInfo   : BOOL := FALSE;                              // 是否请求响应信息
    bRequestProblemInfo    : BOOL := TRUE;                               // 是否请求问题信息

这里的关键不是字段多，而是职责清晰：连接参数只影响建链，发布订阅参数只在对应命令上升沿进入报文构建。

TCP 通了不代表 MQTT 会话已建立。只有 CONNACK 成功后，bMqttConnected 才应该置位。

这一篇最该记住的几句话

FB_MqttClient 不是报文函数集合，而是一个由 eState 驱动的通信状态机。

本篇结论

FB_MqttClient 是状态机驱动的协议功能块。调试时不要只看输入命令，要同时看 eState、TCP 状态、MQTT 状态、错误文本和事务表。
