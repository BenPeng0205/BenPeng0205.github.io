# 技术路线与工具链

## 当前路线

ControlRookie 网站采用“Open Design 高保真原型先行，再迁移到零依赖静态前端工程”的路线。当前阶段不安装任何新依赖，用原生 HTML/CSS/JS 固化已验收视觉，并补齐可静态发布所需的 SEO、发布资产、内容数据和综合验收脚本。

## 工具分工

| 工具 | 定位 | 使用边界 |
| --- | --- | --- |
| Open Design | 主设计工具 | 首页、作品墙、核心页面高保真重构与迭代。 |
| ForAI 原站 HTML | 视觉标尺 | 只用于理解字体、间距、结构和截图对照，不作为正式开发底座。 |
| FrontendDesign | 辅助工具 | 后续可用于产品区、博客区、联系区等内容密集模块的备选探索。 |
| Chromium/Playwright | 验收工具 | 辅助桌面浏览器验证和桌面截图，检查首屏、导航、搜索与主视觉可见性。 |
| Git | 版本管理 | 每轮通过验收后提交，过程产物不长期保留。 |

## 为什么不直接改原站 HTML

ForAI 原站 HTML 是 Framer 编译产物，短期像素相似度高，但不适合作为正式项目底座：

- 类名和结构由 Framer 自动生成，不利于长期维护。
- 依赖远程 Framer 脚本、字体、模块和动画状态。
- 修改内容体系、移动端和产品板块时成本高。
- 后续组件化、SEO、博客、产品数据结构都会受限。

## 为什么选择 Open Design

- 输出更接近可维护 HTML/CSS。
- 便于逐屏校准 ForAI 的视觉体系。
- 便于后续迁移到正式前端工程。
- 可控制动画复杂度，符合静态高保真加简单动效的维护目标。

## 当前工程规则

1. 原型文件继续作为视觉基线，不作为日常开发入口。
2. 日常开发入口为 `src/app/index.html`，其控件注释必须保留，方便后续手动精细调整。
3. 视觉参数集中在 `src/styles/site.css`；未经明确设计调整，不得改动黑白灰、`#f1f1f1`、字号层级、留白和卡片比例。
4. 资源、文章、产品和联系方式统一维护在 `src/content/site-data.js`；`src/lib/site.js` 负责双语、搜索索引生成和轻量动效。
5. 未经用户同意，不安装 Astro、Next.js、npm 包或其它新依赖。若必须安装，先说明安装路径、用途和预计体积。

## 后续验收规则

1. 每次首页视觉调整后必须用桌面浏览器实际打开页面，优先 Edge。
2. 需要截图时只生成桌面端截图作为辅助证据。
3. 检查照片占位可见、PLC/AI/Code/Share 一屏内可见、标题不裁切、导航不溢出、搜索可展开并返回匹配结果。
4. 中文文档写入后必须做 UTF-8 乱码检查。
5. 移动端验收放到正式响应式阶段单独执行，不作为当前高保真首页主验收依据。
6. 工程化修改后必须运行：

```powershell
& 'E:\ai-relavant\ben-blog-cli\.venv\Scripts\python.exe' 'scripts\verify_production_ready.py'
& 'E:\ai-relavant\ben-blog-cli\.venv\Scripts\python.exe' 'scripts\verify_visual_parity.py'
& 'E:\ai-relavant\ben-blog-cli\.venv\Scripts\python.exe' 'scripts\verify_static_app.py'
```

## 生产级验收范围

- 视觉：桌面视觉回归低于 1% 差异阈值，避免工程化破坏 ForAI 底座。
- 功能：搜索、双语、锚点、素材加载、键盘 Escape 收起搜索均通过。
- SEO：title、description、canonical、Open Graph、Twitter Card、结构化数据、robots、sitemap、manifest 均存在。
- 内容：文章、产品、资源、联系方式统一由内容数据源维护。
- 编码：中文文档和源码必须通过 UTF-8 乱码检查。
- 版本：提交时只暂存 `projects/controlrookie-website` 下文件。

## GitHub Pages 临时发布

- 临时域名：`https://benpeng0205.github.io/`
- 构建脚本：`scripts/build_static_site.py`
- 构建产物：`dist/`
- 发布配置：`.github/workflows/deploy-pages.yml`

发布要求：

1. GitHub 仓库建议使用 `BenPeng0205/benpeng0205.github.io`，这是 GitHub Pages 用户站点的根域名仓库。
2. 仓库 Pages 设置选择 GitHub Actions 作为发布源。
3. 推送 `master` 或 `main` 后，Actions 会运行构建脚本并发布 `dist/`。
4. 正式域名购买后，替换 `https://benpeng0205.github.io/`，重新生成 sitemap，并重新运行生产级验收。
